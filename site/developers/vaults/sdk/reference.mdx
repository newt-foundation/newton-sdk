## Package

```bash
pnpm add @newton-xyz/newton-shield-sdk viem zod
```

`viem` and `zod` are peer dependencies. Vendor SDKs, such as `@morpho-org/blue-sdk-viem`, are also peer dependencies so applications only install the vault integrations they use.

## Supported Chains

| Chain | Chain ID |
| --- | --- |
| Ethereum | `1` |
| Base | `8453` |
| Sepolia | `11155111` |

Calling `createShield` on an unsupported chain is expected to error before sending a transaction.

## `createShield`

Creates or attaches to a Shield for a curator and vault. A newly created ShieldClient requires you to call `setParams` and `uploadSecrets` to configure it.

```typescript
const shield = await createShield({
  apiKey,
  policyClientAddress?,
  walletClient,
  rpc,
  env?,
  gatewayUrl?,
  pack,
  vault,
  version?,
  expectedParams?,
  skipPolicyBindingCheck?,
  mustDeploy?,
  bypassDelaySeconds?,
  attestationExpiration?,
  wsRpcUrl?,
})
```

### Parameters

| Field | Description |
| --- | --- |
| `apiKey` | The Newton Protocol Gateway api key. Create one for your project on the [dashboard](https://dashboard.newton.xyz/). |
| `policyClientAddress?` | The address of the shield contract if attaching to a known, already deployed instance. Leave empty to deploy a Shield if needed or attach to an existing one based on the creation parameters: owner, vault, and version. |
| `walletClient` | The Viem client for the signer address that will own the shield and sign transactions for it. The address of the wallet client is one of the deploy params that determines the shield address. |
| `rpc` | The rpc url for sending transactions, polling chain, and other viem PublicClient actions. |
| `env?` | The Newton Gateway environment. Leave empty for the default of `prod`. |
| `gatewayUrl?` | The Newton Gateway url, changeable for testing. Leave empty for the apropriate protocol url for the chosen `env`. |
| `pack` | Policy pack used for the vault. See the [policy pack repo](https://github.com/newt-foundation/newton-policy-packs/) for more details. |
| `vault` | The vault address to be managed. The vault address is one of the deploy params that determines the shield address. |
| `version?` | A nonce that is only used for determining the deployment address. Leave empty for `version = 0`. |
| `expectedParams?` | The typed policy params given the chosen policy pack that you expect the shield instance you are attaching to to have. Will error if the params don't match what is expected. Leave empty to instead just check that the existing params are correctly typed for the chosen pack and the correct policy address is used. Only applicable to attaching to an existing instance, not a new deploy. |
| `skipPolicyBindingCheck?` | Flag for skipping the policy pack/contract check entirely and blindly attaching to the existing deploy (use with caution!). Only relevant for attaching to existing deploys. |
| `mustDeploy?` | Flag to error if the call to `createShield` attempts to attach to an existing deploy. Use to avoid attaching to an unexpected existing insance when you intend to deploy. |
| `bypassDelaySeconds?` | Configuration parameter for the shield owner takeover procedure in the case of an outage in the protocol. Default of 1 week delay, minimum of 1 day. |
| `attestationExpiration?` | Configuration parameter for the policy approvals. Denotes the number of blocks that an approved policy action is valid for. Default of 50 blocks. |
| `wsRpcUrl` | An additional websocket url for your rpc provider to allow for `ATTESTATION` based executions. These executions need to poll for Newton Protocol aggregator submission of the TaskResponse on chain. These executions are slightly slower but save significant gas. |

### Return Value

Calling `createShield` returns a `ShieldClient` object with these properties.

| Property | Description |
| --- | --- |
| `pack` | The policy pack object that it was configured with. Useful as the policy pack method functions help with the typing of params and secrets objects. |
| `policyClientAddress` | The address of the shield contract created or attached to by the client. |
| `encodeParams` | A helper function for encoding the policy params into hex. Useful for comparing to the configuration on chain. |
| `setParams` | Outlined below, this function is one of the main configuration steps for the policy after the Shield contract has been deployed. |
| `uploadSecrets` | Outlined below, many policies require api secrets for accessing data during policy evaluation. This function encrypts and stores these values in the Newton Policy Gateway instead of on chain. |
| `sendCall` | Outlined below, this function is the entrypoint for policy evaluation and intent execution. |

## `setParams`

Configures a newly created Shield contract or reconfigures an existing one by calling the contract function on the Shield contract with the serialized paramters. Uses the `attestationExpiration` value from the shield client creation.

```typescript
await shield.setParams({
  params,
})
```

### Parameters

| Field | Description |
| --- | --- |
| `params` | The typed paramter object from the policy pack used. Import the typing of this object from the relevant npm package for the policy you are using. |

## `uploadSecrets`

Uploads a value for the required api secrets used by the chosen policy pack. The value is encrypted in the sdk and uploaded to the Newton Protocol Gateway where it is stored securely and only accessed to enable policy evaluation.

```typescript
await shield.uploadSecrets({
  secrets,
})
```

### Parameters

| Field | Description |
| --- | --- |
| `secrets` | The typed secrets object from the policy pack used. Import the typing of this object from the relevant npm package for the policy you are using. |

## Shield Runtime

The Shield runtime turns a typed call into:

1. Intent construction
2. Newton AVS policy evaluation
3. Policy attestation retrieval
4. Onchain execution through the Shield contract

Vendor modules are expected to be runtime-pluggable with a viem-style `.extend(...)` pattern.

```typescript
const shield = await createShield(config).extend(morpho())
```

## Morpho Module

The Morpho module is expected to expose typed vault-management actions such as:

```typescript
await shield.morpho.reallocate(args)
await shield.morpho.setCap(args)
```

## `sendCall`

For vendors or actions without a first-class module, use the `sendCall` function. This function requires you to encode the calldata for the delegated contract call yourself. The wasmArgs parameter is a typed object defined by the policy pack used and informs the policy about what must be evaluated for this specific call.

```typescript
await shield.sendCall(
  {
    to: '0xTargetAddress',
    data: '0xCalldata',
    functionSignature: 'setCap(bytes32,uint184)',
    wasmArgs: {
      vaultAddress: '0xVaultAddress',
      network: 'base',
    },
  }.
  submissionMode = "DIRECT",
  timeoutMs = 30_000,
)
```

When using `sendCall`, the integration owns the intent integrity for the target call. Use `submissionMode` of `"ATTESATION"` if you have provided a `wsRpcUrl` when creating your ShieldClient and you wish to save gas at the cost of speed of execution.

## Policy Packs

Policy packs are developed in [newt-foundation/newton-policy-packs](https://github.com/newt-foundation/newton-policy-packs) and are expected to be available as subpath imports:

```typescript
import { vaultRiskPack } from '@newton-xyz/newton-shield-sdk/packs/vault-risk'
```

Each pack provides typed params and helper logic for fetching freshness-bound policy inputs.

## Errors

The target SDK error hierarchy includes:

| Error | When it occurs |
| --- | --- |
| `InvalidConfigurationError` | Error for when a parameter for `createShield` is invalid. |
| `ParamMismatchError` | The expected params or param typing does not match the configuration of an attached Shield contract. |
| `PolicyMismatchError` | The expected policy from the pack does not match the configuration of an attached Shield contract. |
| `ShieldExecutionError` | The execution of the intent on chain encounters an error. |
| `DecodedContractError` | The error found when simulation of execution reverts. |
| `PolicyDeniedError` | Newton evaluates the policy and denies the action. |
| `NewtonTaskEvaluationError` | Newton evaluation fails to evaluate the policy correctly. |
| `AttestationTimeoutError` | The SDK does not receive an attestation in time. |
| `ShieldExecutionError` | The Shield contract execution fails. |
| `GatewayHttpError` | The request fails to reach the Newton Gateway. |
| `ShieldDeploymentNotFoundError` | ShieldClient creation tries to use a chain without Shield deployments. |
| `UnsupportedChainError` | ShieldClient creation tries to use a chain that is unsupported by the Newton Protocol. |

## Browser Safety

The target SDK core is browser-safe and avoids `node:*` imports.
