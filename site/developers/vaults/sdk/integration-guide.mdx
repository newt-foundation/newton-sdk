The Shield SDK is designed to sit beside the Morpho SDK rather than replace it. Your app prepares the same typed vault-management action it would normally send through Morpho, then routes execution through a Shield that requires a Newton attestation.

## 1. Install Packages

When available, install the Shield SDK and the peer dependencies your app uses:

```bash
pnpm add @newton-xyz/newton-shield-sdk viem zod @morpho-org/blue-sdk-viem
```

## 2. Create a Wallet Client

```typescript
import { createWalletClient, http } from 'viem'
import { base } from 'viem/chains'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount(process.env.CURATOR_PRIVATE_KEY as `0x${string}`)

const walletClient = createWalletClient({
  account,
  chain: base,
  transport: http(process.env.BASE_RPC_URL),
})
```

## 3. Create or Attach to a Shield

```typescript
import { createShield } from '@newton-xyz/newton-shield-sdk'
import { vaultRiskPack } from '@newton-xyz/newton-shield-sdk/packs/vault-risk'
import { morpho } from '@newton-xyz/newton-shield-sdk/vendors/morpho'

const shield = await createShield({
  apiKey: "YOUR_NEWTON_API_KEY",
  walletClient,
  rpc: process.env.BASE_RPC_URL,
  pack: vaultRiskPack,
  vault: yourVaultAddress,
  mustDeploy: true,
}).extend(morpho())
```

`createShield` is intended to be idempotent. It predicts a deterministic Shield address from the curator and vault, attaches to an existing Shield when parameters match, or deploys one when needed. In the above example, the call requries the function to deploy the contract. If instead attaching to an existing deployed instance, it is adviseable to make sure it has the expected policy params as so:

```typescript
import { createShield } from '@newton-xyz/newton-shield-sdk'
import { vaultRiskPack } from '@newton-xyz/newton-shield-sdk/packs/vault-risk'
import { morpho } from '@newton-xyz/newton-shield-sdk/vendors/morpho'

const shield = await createShield({
  apiKey: "YOUR_NEWTON_API_KEY",
  // not required but adviseable. fully determined by walletClient, vault, and version params
  policyClientAddress: expectedPolicyClientAddress,
  walletClient,
  rpc: process.env.BASE_RPC_URL,
  pack: vaultRiskPack,
  vault: yourVaultAddress,
  version: 1,
  expectedParams: {
    // typed policy params defined by the chosen policy pack
    ...params
  },
}).extend(morpho())
```

## 4. Configure your Shield contract.

```typescript
await shield.setParams({
  risk_score_floor: 85,
  tvl_drawdown_24h_max_pct: 5,
  tvl_drawdown_7d_max_pct: 20,
  etc...
})
await shield.uploadSecrets({
  api_key: "MY_API_KEY",
  rpc_url: "MY_RCP_URL",
  etc...
})
```

After creating a Shield contract for the first time or if you want to tweak the parameters later, you must configure the policy defined in the policy pack you are using to fit your needs. These parameters apply to each evaluation and represent a global configuration of the security of your vault.

## 5. Execute a Morpho Action

```typescript
import { parseUnits } from 'viem'

await shield.morpho.submitCap({
  marketId: '0xMorphoMarketId',
  cap: parseUnits('5000000', 6),
  wasmArgs: {
    vaultAddress: '0xVaultAddress',
    network: 'base',
    symbol: 'USDC',
  },
})
```

In this example, `submitCap` is only forwarded to MetaMorpho if Newton approves the intent. If the Vaults.fyi or oracle-divergence checks fail, the SDK raises a typed policy error instead of sending the vault-management transaction.

## 6. Execute a generic vault action

```typescript
import { encodeFunctionData } from 'viem'
import { EulerEarnAbi } from './constants/abis'

await shield.sendCall({
  to: '0xEulerEarnContract',
  data: encodeFunctionData({
    abi: EulerEarnAbi,
    functionName: 'reallocate',
    args: [marketAllocations],
  }),
  functionSignature: 'reallocate((bytes32,uint256)[])',
  wasmArgs: {
    vaultAddress: '0xVaultAddress',
    network: 'base',
    symbol: 'USDC',
  },
})
```

In this example, the same policy checks are performed as before, the only difference being explicit encoding of the function calldata to be performed if the policy check succeeds.

## 7. Handle Policy Errors

```typescript
import { PolicyDeniedError } from '@newton-xyz/newton-shield-sdk'

try {
  await shield.morpho.submitCap({
    marketId: '0xMorphoMarketId',
    cap: parseUnits('5000000', 6),
    wasmArgs: {
      vaultAddress: '0xVaultAddress',
      network: 'base',
      symbol: 'USDC',
    },
  })
} catch (error) {
  if (error instanceof PolicyDeniedError) {
    console.error('Vault action denied by policy', error)
    return
  }

  throw error
}
```
